How to Run the
Project on Your Computer

A friendly, plain-English guide to getting CodeCards up and running on your own machine — no prior experience required.

You can do this. Every single step is written out in full, with explanations for what each step actually does and why it matters. If something looks unfamiliar, keep reading — it will be explained.
What this guide does: It takes you from a fresh computer to a fully running version of CodeCards in about 10–15 minutes. You will install one program (Node.js), copy one line of text into a config file, type two commands in a terminal, and open a file in your browser. That is genuinely all there is to it. The database is already set up and running in the cloud — you just need to connect to it with a short address that will be provided to you.

1 Install Node.js

Node.js is a program that lets your computer run JavaScript code outside of a browser. The CodeCards backend is written in JavaScript, so your computer needs Node.js installed before it can understand and execute that code. Think of it like installing a specific app that can "speak" the language the backend is written in.

If you already have Node.js installed, you can skip ahead. To check, open a terminal (explained in Step 3 below) and type node -v then press Enter. If you see a version number like v20.14.0, you are all set. If you see an error saying the command was not found, proceed with the installation below.

1

Open your web browser and go to nodejs.org. You will see two big download buttons. Click the one labelled LTS — this stands for "Long-Term Support" and is the stable, recommended version. A file will download to your Downloads folder.

2

Open the downloaded file (it will be named something like node-v20.x.x-x64.msi on Windows). An installer window will appear. Click Next through each screen and accept the default settings — you do not need to change anything. Click Install when prompted, and then Finish when it is done.

3

Verify the installation worked. Open a terminal (see Step 3 for instructions on how to do that) and type the following, then press Enter: 

node -v

You should see a version number printed. Then type this and press Enter: 

npm -v

You should see another version number. Both of these working means Node.js and its package manager (npm) are correctly installed.

What is npm? npm stands for "Node Package Manager." It is a tool that comes bundled with Node.js and lets you download and manage the extra code libraries that a project depends on. You will use it in Step 5.

2 Get the Project Files

You need a copy of the CodeCards project files on your computer. These will either be shared with you as a ZIP file or via a shared folder. Either way, the important thing is knowing exactly where they end up on your computer.

If you received a ZIP file, right-click it and choose Extract All (on Windows). Choose a sensible location — somewhere like your Desktop or your Documents folder works fine. After extracting you should see a folder called codecards. Open it and you should see two sub-folders: backend and frontend. That is the correct structure.

Remember where you put it. You will need to navigate into this folder using a terminal in the next step. If you are not sure where you extracted the files, right-click the backend folder and choose "Properties" (Windows) — it will show you the full path.

3 Open a Terminal

A terminal (also called a "command prompt" or "command line") is a text-based window where you type instructions directly to your computer. It can sound intimidating if you have never used one, but for this project you only need to type a handful of short commands, all of which are spelled out for you exactly.

Opening a terminal on Windows

The easiest method: hold Shift and right-click inside the backend folder in Windows Explorer (the file browser). A menu will appear, and one of the options should say "Open PowerShell window here" or "Open in Terminal". Click that. A terminal window will open already pointed at the right folder — perfect.

Alternatively, press the Windows key, type cmd, and press Enter to open Command Prompt. Or type powershell for PowerShell. Both work. Once it is open you will need to navigate to the backend folder manually (see below).

Navigating to the backend folder

If your terminal did not open directly in the backend folder, you need to tell it where to go. The command for this is cd which stands for "change directory" — another word for folder. Type it like this, replacing the path with wherever you actually put the project: 

cd C:\Users\YourName\Desktop\codecards\backend

Press Enter. The prompt should now show the path ending with \backend. You are in the right place.

Quick tip: You do not have to type the full path by hand. In Windows Explorer, click the address bar at the top of the window (the part showing the folder path), select all the text, and copy it. Then in your terminal type cd (with a space) and paste the copied path.

4 Set Up the Configuration File

Inside the backend folder there is a file called .env. This is a simple text file that holds private configuration values for the project — things like the address of the database and a secret key used to secure user logins. The reason these live in a separate file rather than inside the code itself is so that they are never accidentally shared or uploaded to GitHub.

You may not see the .env file in Windows Explorer by default because files whose names start with a dot are sometimes hidden. If that is the case, open the folder in VS Code instead — it will show the file in the file tree on the left. You can also open VS Code's file explorer and see it there.

Open the .env file in any text editor (VS Code, Notepad, Notepad++ — anything works). It will look something like this: 

# Server
PORT=5000
NODE_ENV=development

# Database
MONGODB_URI="put the uri here"

# Authentication
JWT_SECRET=OIJoijfowiejojOEF93FoijfoIJWEOFJ230fj09FJ2093FJODIFJOIJEFojoiw009JFoiji
JWT_EXPIRES_IN=7d

The only line you need to update is the MONGODB_URI line. You will be given a database address (URI) by the project lead — it is a long string of text that starts with mongodb+srv://. Replace the placeholder text with that string, keeping the quotation marks around it. The finished line should look something like this (your actual URI will be different): 

MONGODB_URI="mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/codecards"
Important: Do not add any spaces before or after the equals sign. Do not change the quotes. Copy the URI exactly as given — if even one character is wrong, the server will not be able to connect to the database. Triple-check it.
Everything else in the .env file is already set correctly. The PORT, JWT_SECRET, and JWT_EXPIRES_IN values are already filled in and do not need to be touched. Only the MONGODB_URI needs to be updated with the address you receive.

5 Install the Project's Packages

The backend relies on a number of code libraries written by other developers — things like the database connector, the security tools, and the web server framework. These libraries are not stored inside the project folder itself (to keep the project size small), but there is a list of everything that is needed in a file called package.json. The npm install command reads that list and downloads everything automatically.

In your terminal (which should still be open in the backend folder from Step 3), type the following and press Enter: 

npm install

You will see a lot of text scroll by as npm downloads the packages. This is normal. It typically takes 30–60 seconds depending on your internet speed. When it is done, the terminal will return to a blank prompt (just a blinking cursor waiting for input), and a new folder called node_modules will have appeared inside the backend folder. That folder contains all the downloaded libraries.

You only need to do this once. Unless someone adds a new package to the project (in which case the package.json will change), you do not need to run npm install again on subsequent runs. Just go straight to starting the server.

6 Start the Backend Server

Now you are ready to start the backend. In your terminal, type the following and press Enter: 

npm start

The server will start up and attempt to connect to the database. If everything is configured correctly, you will see output that looks like this:

Expected terminal output — success
MongoDB connected: cluster0.xxxxx.mongodb.net
Server running on port 5000

Those two lines mean everything is working. The first tells you the database connection was successful. The second tells you the server is running and listening on port 5000, which is the "door number" your browser will use to talk to it.

The terminal window must stay open while you are using the app. The server runs as long as that terminal window is open. Do not close it. If you accidentally close it, the website will stop working and you just need to run npm start again.

If you see an error instead of the success message, the most likely cause is the MONGODB_URI in your .env file. Go back to Step 4 and double-check that the URI is copied exactly, with no extra spaces or missing characters. The troubleshooting table at the end of this guide covers the most common problems.

7 Open the Website in Your Browser

The frontend of CodeCards is made of plain HTML files. Unlike the backend, it does not need to be "started" — you just open it directly in your browser, the same way you would open any file on your computer.

Open your file browser (Windows Explorer), navigate to the codecards folder, then go into the frontend sub-folder. You will see a file called index.html. Double-click it. It will open in your default web browser and you will see the CodeCards landing page.

Using VS Code? If you have the Live Server extension installed in VS Code (it is free and very popular), right-click index.html in VS Code's file explorer and choose "Open with Live Server". This is slightly better because Live Server automatically refreshes the page when you make changes to the HTML or CSS — handy if you are developing.

8 Try the App End to End

Everything should now be running. Here is a quick walkthrough of the full user journey to confirm that everything is working correctly from the database all the way to the screen.

1

On the landing page, click Sign Up. Fill in a username, email address, and a password (at least 8 characters). Click Create Account. If the server and database are working, you will be taken to your dashboard immediately. If you see an error, check the terminal window for clues.

2

On the dashboard, click + New Deck. Give your deck a title (e.g. "JavaScript Basics"), choose a language from the dropdown, and optionally add a description. Click Create Deck. The new deck card should appear in the grid immediately.

3

Click Open on your new deck. You will see the deck detail page. Click + Add Card, type a question in the Front field and an answer in the Back field. Try adding at least two or three cards so the quiz is meaningful. Click Add Card each time.

4

Click ▶ Start Quiz. The quiz page will load with your cards shuffled. Click a card (or press Space) to flip it and reveal the answer. Then click Got it or Missed it to move to the next card. After the last card you will see a results screen with your score.

5

Go back to the dashboard (click <CodeCards/> in the top left). Scroll down and you should see your just-completed quiz score in the "Recent Quiz Scores" table. That confirms the score was saved to the database successfully.

If all five of those steps worked, the project is running perfectly. Well done.

9 How to Stop the Server

When you are done working and want to turn the server off, click inside the terminal window and press Ctrl + C at the same time. The server will stop and the terminal will return to a normal prompt. You can then close the terminal window safely.

Next time you want to run the project again, you only need to open a terminal in the backend folder and run npm start — no need to repeat Steps 1 through 5. Those are one-time setup steps.

! If Something Goes Wrong

Below is a table of the most common problems people encounter, what the error message looks like, and exactly what to do about it.

What you see What it means & what to do
MongoDB connection error: Protocol and host list are required The MONGODB_URI in your .env file is wrong or missing. Open .env, check that the URI is pasted in full with no extra @ signs, spaces, or missing quotes around it.
querySrv ECONNREFUSED or ENOTFOUND Your computer cannot reach the database server. This is almost always a network/internet issue — check that you are connected to the internet. If you are on a strict corporate or school Wi-Fi, it may be blocking the connection. Try a different network (mobile hotspot, home Wi-Fi) to confirm.
MongoServerError: bad auth The username or password embedded inside the URI is wrong. Ask the project lead to re-send you the correct URI and paste it fresh.
Error: Cannot find module '...' The packages have not been installed yet, or something went wrong during installation. Make sure you are inside the backend folder in your terminal, then run npm install again.
'node' is not recognized as a command Node.js is either not installed or the installer did not add it to your system's PATH. Re-run the Node.js installer from nodejs.org. After it finishes, close and reopen your terminal and try again.
The browser shows a blank page or an error when opening index.html This usually means the backend server is not running. Switch to your terminal and check that you can see the "Server running on port 5000" message. If not, run npm start again.
Login or register shows "Network Error" or "Failed to fetch" The frontend is trying to contact the backend but cannot reach it. Make sure the terminal with npm start is still open and shows no errors. If you are using a browser like Firefox or Chrome, also check that it is not blocking the request for security reasons (look in the browser console for clues).
Port 5000 is already in use Another program on your computer is using port 5000. The simplest fix is to change the port number: open .env, change PORT=5000 to PORT=5001, save the file, and run npm start again. Then also update the API_BASE line in frontend/js/api.js to use :5001 instead.

✓ Quick-Reference Checklist

Once you have completed the initial setup, use this checklist every time you want to run the project to make sure everything is in order before you begin.

  • Node.js is installed — confirmed with node -v
  • The MONGODB_URI in backend/.env is filled in with the provided address
  • npm install has been run at least once in the backend folder
  • Terminal is open inside the backend folder
  • Server is started with npm start and shows "Server running on port 5000"
  • frontend/index.html is open in the browser
